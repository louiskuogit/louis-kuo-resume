System Design

從網站到營運後台的架構思維

真實落地時，前台報名頁、會員登入、付款流程、票券庫存、客服後台與營運報表會拆成清楚的服務邊界； 尖峰活動則需要 CDN、快取、排隊、容量壓測、監控告警與可回溯的操作紀錄。

Frontend

活動頁、報名表、會員中心、票券資訊與付款流程。

API & Queue

訂單狀態、庫存鎖定、付款回呼、排隊與重試機制。

Cloud Operations

GKE / VM、CDN、快取、監控告警、自動擴展與備援。

Management

營運報表、客服事件、權限控管、稽核紀錄與改善追蹤。

Why This Matters

這個作品能幫履歷加分的地方

對票務 / 活動平台：能理解售票尖峰、庫存鎖定、付款、客服與營運報表，不只是做畫面。

對資訊主管職：能把業務流程、系統穩定度、跨部門協作與管理指標整合在一起。

對 DevOps / SRE 職缺：能說明尖峰流量、容量規劃、監控告警、Runbook 與故障應變。
